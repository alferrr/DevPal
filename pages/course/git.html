Git Fundamentals

Git Course

Introduction

In this unit, you'll learn how developers manage and track changes in code using Git. Git is the most widely used version control system in the world and is an essential tool for professional development.

Whether you're building projects alone or collaborating with teams, Git allows you to track changes, revert mistakes, create new features safely, and maintain a clean history of your work.

By the end of this course, you'll understand how Git works, how to track changes in your projects, and how to work with remote repositories like GitHub.

What Git Really Is

Version Control

Git is a version control system. It keeps track of changes in files over time.

  • Tracks changes in code
  • Allows you to revert mistakes
  • Maintains a full project history
  • Supports collaboration between developers

Why Developers Use Git

  • Prevents losing work
  • Allows experimentation safely
  • Makes teamwork easier

Installing Git

Getting Git

Git runs from the command line and works on all major operating systems.

  • Windows
  • macOS
  • Linux

Verify Installation

git --version

If Git is installed correctly, this command will show the installed version.

Repositories

What is a Repository?

A repository (repo) is a project folder that Git tracks.

  • Stores your files
  • Tracks changes
  • Maintains commit history

Creating a Repository

git init

This command turns the current folder into a Git repository.

Tracking Files

Staging Files

Git uses a staging area to prepare files before saving them to history. 

git add filename

To add all files:

git add .

Checking Status

git status

This command shows which files are tracked, modified, or staged.

Commits

What is a Commit?

A commit is a saved snapshot of your project.

  • Records changes
  • Includes a message describing what changed
git commit -m "Initial project setup"

Commit Best Practices

  • Write clear messages
  • Keep commits focused
  • Commit often

Viewing Project History

Git Log

git log

This command shows the full commit history of your project.

  • Commit IDs
  • Author information
  • Commit messages
  • Timestamps

Branches

What Are Branches?

Branches allow you to work on new features without affecting the main project.

  • Create experimental features
  • Fix bugs separately
  • Develop safely
git branch feature-name
git checkout feature-name

GitHub and Remote Repositories

What is GitHub?

GitHub is a platform that hosts Git repositories online.

  • Backup your code
  • Share projects
  • Collaborate with other developers

Connecting a Remote Repository

git remote add origin repository-url
git push -u origin main

Basic Git Workflow

Most developers follow this workflow:

  1. Edit files
  2. Stage changes
  3. Commit changes
  4. Push to GitHub

This creates a reliable history of your project.

Collaboration with Git

How Teams Work With Git

In real development teams, multiple developers work on the same project. Git allows everyone to work safely without overwriting each other's code.

Most teams follow a workflow where new features are built on separate branches and then merged into the main project using pull requests.

Step 1 — Update Your Local Main Branch

Before starting new work, you should always make sure your local main branch is up to date with the remote repository. 

git checkout main
git pull origin main

This ensures you're starting from the latest version of the project.

Step 2 — Create a Feature Branch

Instead of working directly on main, developers create a new branch for each feature or fix. 

git checkout -b feature-login-page

This keeps the main project stable while new work is being developed.

Step 3 — Make Changes and Commit

After making changes to the project, stage and commit your work. 

git add .
git commit -m "Add login page layout"

Commits create snapshots of your progress and help track changes.

Step 4 — Push Your Branch

Once your work is ready to share, push your branch to the remote repository. 

git push origin feature-login-page

Your branch will now appear on platforms like GitHub.

Step 5 — Create a Pull Request

On GitHub, you open a Pull Request (PR) to propose merging your feature into the main branch.

  • Explain what your changes do
  • Provide screenshots if necessary
  • Link related issues

A pull request allows team members to review the code before it becomes part of the main project.

Step 6 — Code Review and Merge

Other developers review the changes and may request improvements.

  • Check code quality
  • Ensure the feature works correctly
  • Suggest improvements

Once approved, the pull request is merged into the main branch, and the feature becomes part of the project.

The Full Collaboration Workflow

git checkout main
git pull origin main
git checkout -b feature-name
git add .
git commit -m "Describe your changes"
git push origin feature-name

Then create a Pull Request on GitHub and wait for review before merging.

Git Best Practices

  • Commit often
  • Write meaningful commit messages
  • Use branches for new features
  • Keep repositories organized
  • Push work regularly

These habits make collaboration and project maintenance much easier.
