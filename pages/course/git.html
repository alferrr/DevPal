Git Fundamentals

Git Course

Introduction

In this unit, you'll learn how developers manage and track changes in code using Git. Git is the most widely used version control system in the world and is an essential tool for professional development.

Whether you're building projects alone or collaborating with teams, Git allows you to track changes, revert mistakes, create new features safely, and maintain a clean history of your work.

By the end of this course, you'll understand how Git works, how to track changes in your projects, and how to work with remote repositories like GitHub.

What Git Really Is

Version Control

Git is a version control system. It keeps track of changes in files over time.

  • Tracks changes in code
  • Allows you to revert mistakes
  • Maintains a full project history
  • Supports collaboration between developers

Why Developers Use Git

  • Prevents losing work
  • Allows experimentation safely
  • Makes teamwork easier

Installing Git

Getting Git

Git runs from the command line and works on all major operating systems.

  • Windows
  • macOS
  • Linux

Verify Installation

git --version

If Git is installed correctly, this command will show the installed version.

Repositories

What is a Repository?

A repository (repo) is a project folder that Git tracks.

  • Stores your files
  • Tracks changes
  • Maintains commit history

Creating a Repository

git init

This command turns the current folder into a Git repository.

Tracking Files

Staging Files

Git uses a staging area to prepare files before saving them to history. 

git add filename

To add all files:

git add .

Checking Status

git status

This command shows which files are tracked, modified, or staged.

Commits

What is a Commit?

A commit is a saved snapshot of your project.

  • Records changes
  • Includes a message describing what changed
git commit -m "Initial project setup"

Commit Best Practices

  • Write clear messages
  • Keep commits focused
  • Commit often

Viewing Project History

Git Log

git log

This command shows the full commit history of your project.

  • Commit IDs
  • Author information
  • Commit messages
  • Timestamps

Branches

What Are Branches?

Branches allow you to work on new features without affecting the main project.

  • Create experimental features
  • Fix bugs separately
  • Develop safely
git branch feature-name
git checkout feature-name

GitHub and Remote Repositories

What is GitHub?

GitHub is a platform that hosts Git repositories online.

  • Backup your code
  • Share projects
  • Collaborate with other developers

Connecting a Remote Repository

git remote add origin repository-url
git push -u origin main

Basic Git Workflow

Most developers follow this workflow:

  1. Edit files
  2. Stage changes
  3. Commit changes
  4. Push to GitHub

This creates a reliable history of your project.

Collaboration with Git

Working With Other Developers

One of Git’s greatest strengths is enabling developers to collaborate on the same project without overwriting each other’s work.

  • Multiple developers can work on the same repository
  • Each developer can create their own branches
  • Changes can be reviewed before merging

Pulling Changes

When working with a team, you should regularly pull updates from the shared repository. 

git pull

This command fetches the latest changes from the remote repository and updates your local project.

Pull Requests

On platforms like GitHub, developers typically submit their changes through a pull request.

  • Propose new features or fixes
  • Allow team members to review code
  • Ensure code quality before merging

Code Reviews

Reviewing code is an important part of teamwork. It helps catch bugs, improve code quality, and share knowledge across the team.

  • Check logic and readability
  • Suggest improvements
  • Ensure project standards are followed

Collaboration is a core part of modern development, and Git makes it possible for teams around the world to build software together.

Git Best Practices

  • Commit often
  • Write meaningful commit messages
  • Use branches for new features
  • Keep repositories organized
  • Push work regularly

These habits make collaboration and project maintenance much easier.

Git is a core skill used by nearly every professional developer. Mastering it will help you build projects confidently and work with teams more effectively.
